# 《赛博机甲 SebeJJ》视觉风格指南

## 1. 整体美术方向

### 核心理念
深海中的赛博朋克——在无尽黑暗的海底，机甲的霓虹光芒是唯一的光源。营造孤独、神秘、科技感与压迫感并存的氛围。

### 关键词
- 深海恐惧 (Thalassophobia)
- 赛博朋克 (Cyberpunk)
- 霓虹美学 (Neon Aesthetic)
- 机械美学 (Mecha Aesthetics)
- 资源掠夺 (Resource Extraction)

---

## 2. 配色方案

### 主色调
| 颜色名称 | 色值 | 用途 |
|---------|------|------|
| 深海黑 | #0A0A0F | 背景主色 |
| 深渊蓝 | #0D1B2A | 深海渐变 |
| 赛博青 | #00F0FF | 主强调色，能量、科技 |
| 霓虹粉 | #FF00A0 | 危险、警告、敌人 |
| 电光紫 | #8B5CF6 | 稀有资源、高级装备 |
| 生物荧光绿 | #39FF14 | 生命、生物发光 |
| 警报橙 | #FF6B00 | 低血量、紧急状态 |
| 纯白 | #FFFFFF | 高光、文字 |

### 配色应用规则
- **背景层**: 深海黑 → 深渊蓝 渐变
- **前景元素**: 赛博青 + 霓虹粉 作为主霓虹光源
- **稀有/特殊**: 电光紫 用于传说级物品
- **危险提示**: 警报橙 + 霓虹粉

---

## 3. 光影风格

### 光照系统
- **主光源**: 机甲探照灯（锥形光束，边缘衰减）
- **环境光**: 极弱，仅用于区分轮廓
- **自发光**: 所有科技元素带霓虹辉光
- **生物发光**: 深海生物的柔和荧光

### 辉光效果 (Bloom)
- 强度: 1.5-2.0
- 阈值: 0.8
- 扩散: 柔和的光晕

---

## 4. 机甲设计规范

### 视觉层级
| 等级 | 视觉特征 | 颜色标识 |
|-----|---------|---------|
| 基础型 (Mk.I) | 简洁线条，基础装备 | 赛博青 |
| 进阶型 (Mk.II) | 更多装甲板块，武器挂载 | 赛博青 + 白 |
| 精锐型 (Mk.III) | 重型装甲，复杂结构 | 电光紫点缀 |
| 传说型 (Legend) | 独特造型，华丽特效 | 电光紫主色 |

### 机甲结构要素
1. **驾驶舱**: 半透明玻璃，内部微光
2. **推进器**: 发光喷口，粒子拖尾
3. **机械臂**: 可替换的装备接口
4. **核心反应堆**: 胸部发光核心
5. **探照灯**: 头部/肩部照明设备

## 5. UI 设计规范

### 整体风格
- 全息投影 (Holographic)
- 故障艺术 (Glitch Art) 用于伤害/警告
- 极简线条，高对比度

### UI 元素
- **边框**: 1-2px 赛博青线条，直角或45度切角
- **面板**: 半透明深色背景 (rgba(10,10,15,0.85))
- **文字**: 等宽字体，发光效果
- **图标**: 线性图标，霓虹描边

### 字体规范
- **标题**: Rajdhani / Orbitron (科技风)
- **正文**: Share Tech Mono / 思源黑体
- **数字**: 等宽字体，用于数值显示

### 动效规范
- 按钮悬停: 颜色闪烁 + 轻微位移
- 面板展开: 扫描线入场
- 警告: 故障抖动 + 红色闪烁

---

## 6. 背景设计规范

### 深度分层
| 深度 | 背景色调 | 可见度 | 特殊元素 |
|-----|---------|-------|---------|
| 100米 | 深蓝 → 浅蓝 | 高 | 阳光穿透，珊瑚礁 |
| 500米 | 深蓝 → 黑 | 中 | 微光生物，沉船 |
| 1000米 | 近黑 | 低 | 热液喷口，奇异生物 |
| 2000米 | 纯黑 | 极低 | 未知深渊，远古遗迹 |

### 视差层次
1. **前景**: 机甲、近处物体
2. **中景**: 矿脉、小型沉船
3. **远景**: 深海地形、巨大阴影
4. **背景**: 无尽黑暗，偶尔的生物发光

---

## 7. 特效规范

### 粒子效果
- **推进器尾焰**: 赛博青粒子，渐隐拖尾
- **气泡**: 半透明，上升动画
- **火花**: 短寿命，橙黄色
- **能量护盾**: 六边形网格，受击涟漪

### 着色器效果
- **扫描线**: 全息UI必备
- **扭曲**: 水下折射效果
- **发光**: 所有霓虹元素
- **故障**: 伤害/系统错误时使用

## 8. 资源命名规范

```
[类型]_[名称]_[变体]_[状态].[格式]

示例:
- CHR_Mecha_Mk1_Base.png
- CHR_Mecha_Mk1_Damaged.png
- BG_DeepSea_100m_Layer1.png
- UI_Button_Primary_Hover.png
- FX_Scanner_Wave_01.png
- PROP_Shipwreck_Cargo_01.png
```

### 类型前缀
- CHR_: 角色/机甲
- BG_: 背景
- UI_: 用户界面
- FX_: 特效
- PROP_: 道具/场景物件
- ITM_: 物品/资源

## 9. 技术规格

### 分辨率
- 机甲精灵: 256x256 ~ 512x512
- 背景图层: 1920x1080 (可拼接)
- UI元素: 9-slice 可缩放
- 特效: 128x128 ~ 256x256

### 格式
- 精灵: PNG (透明通道)
- 动画: 精灵图集
- UI: PNG + 9-slice 数据
- 特效: PNG 序列或 Shader

### Unity 设置
- PPU (Pixels Per Unit): 100
- Filter Mode: Point (像素风) / Bilinear (平滑)
- Compression: 无压缩或高质量
